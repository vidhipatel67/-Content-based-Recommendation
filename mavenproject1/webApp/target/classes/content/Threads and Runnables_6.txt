<p>In light of the above discussion, a thread is the smallest unit of processing that can be scheduled by an operating system. Therefore, using threads, a programmer can effectively create two or more tasks<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup> that run at the same time. The first call-to-action is to implement a set of tasks that a particular thread would execute. To do so, we require the creation of a <code>Runnable</code> process.</p>
